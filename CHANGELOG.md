# Changelog

すべての注目すべき変更はこのファイルに記録する。

運用ルール（FR-7）: develop → main へのリリース PR を作成する際、`## [Unreleased]`
を `## [x.y.z] - YYYY-MM-DD`（`pubspec.yaml` の新バージョンと一致させる）にリネームし、
先頭に新しい空の `## [Unreleased]` を追加してからマージする。main への push を検知して
CI がこのファイルから該当バージョンのリリースノートを読み取り、Firestore の
`meta/release` に反映する。

## [Unreleased]

## [1.3.1] - 2026-07-14
- 招待リンクを貼り付けて、他の人のカレンダーに参加できるようになりました
- 設定画面からアプリの更新履歴を見られるようになりました

## [1.3.0] - 2026-07-13
- 家族メールの事前登録（allowlist）なしでサインインできるようになりました
- アカウント作成時に、自分だけが参加するカレンダーが自動で作られるようになりました
- カレンダーにオーナーが設定され、カレンダー名の変更やメンバーの管理はオーナーだけが行えるようになりました
- カレンダーからメンバーを外したり、自分が退出したり、オーナーを他のメンバーに引き継げるようになりました

## [1.2.0] - 2026-07-11
- メンバーを選んで、その人の予定だけを絞り込んで表示できるようになりました
- 既存の予定をコピーして、新しい予定を素早く作れるようになりました
- 同じ名前で連続する予定を1本の帯にまとめて見やすく表示するようにしました
- 複数日にまたがる予定を、全幅の帯で見やすく表示するようにしました
- 繰り返しの予定を作成できるようになりました
- アプリアイコンに合わせた和風テーマを追加し、表示テーマを切り替えられるようにしました
- カレンダー編集時のメンバー変更で更新が失われる不具合を修正しました

## [1.1.0] - 2026-07-09
- 複数のカレンダーを作成・切り替えできるようになりました
- 予定編集画面に作成者を表示するようにしました
- 月表示で前後月のセルにもその日の予定が表示されるようになりました
- メンバーの色の選択肢を増やしました
- 日別予定画面の参加者表示・視認性・日付移動の操作性を改善しました
- 予定の開始時刻を変更しても期間の長さが保たれるようにしました
- 複数日にまたがる予定のタイトル表示・月表示での見え方を改善しました
- 日付ピッカーの表記を日本語にしました

## [1.0.0] - 2026-07-08
- Google / Apple アカウントでログインできるようになりました
- カレンダーの月表示に対応し、祝日もハイライト表示されます
- 「YYYY年MM月」をタップすると、スクロールで素早く別の年月に移動できます
- 予定に参加者を複数人設定でき、参加者ごとに色分けして表示されます
- 予定の期間（複数日にまたがる予定）を指定できるようになりました
- 日別の予定一覧でメモの内容をプレビュー表示するようにしました
- 自分が参加する予定を一覧の先頭に表示するようにしました
- 予定の時刻をスクロールで選べるようになりました
- 表示名を設定・変更できるようになりました
- アプリの新しいバージョンが公開されたときに、アプリ内でお知らせするようにしました
- アプリアイコンを新しいデザインに変更しました
