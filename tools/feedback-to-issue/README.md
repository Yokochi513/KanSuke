# フィードバック自動 Issue 化

Google フォームの回答を GitHub Issue（`Yokochi513/KanSuke`）として自動登録する仕組み。
`Code.gs` を Google フォームに紐づく Apps Script として設置する。

すでに `フィードバック` ラベルはリポジトリに作成済み。

## 0. フォーム構成案

`Code.gs` は質問と回答をそのまま Issue 本文に列挙し、**1問目の回答**を
Issue タイトルの手がかりに使う（30文字以内ならそのままタイトルに採用、
それ以外は送信日時をタイトルにする）。そのため1問目は短い一言タイトルにするのが良い。

| # | 質問 | 形式 | 必須 |
| --- | --- | --- | --- |
| 1 | 一言でいうと？（20〜30文字くらい）例:「予定が重複して見える」 | 記述式 | 必須 |
| 2 | 種類 | 選択式: バグ・不具合 / 使いにくい・分かりにくい / こんな機能が欲しい / その他・感想 | 必須 |
| 3 | 詳しく教えてください（何が起きた／どうなってほしいか） | 段落 | 必須 |
| 4 | 使っていた画面・操作（わかれば） | 選択式: カレンダー(月表示) / 予定の作成・編集 / ログイン / 通知 / その他・わからない | 任意 |
| 5 | 使用端末 | 選択式: iPhone / Android / PC(ブラウザ) / その他 | 任意 |
| 6 | お名前（任意・誰からか分かると助かります） | 記述式 | 任意 |

## 1. GitHub Personal Access Token を発行する

1. https://github.com/settings/tokens?type=beta を開く（Fine-grained token）
2. 「Generate new token」
3. Repository access: **Only select repositories** → `KanSuke` を選択
4. Permissions → Repository permissions → **Issues: Read and write** のみ付与
5. 有効期限を設定して発行し、表示されたトークン文字列を控える（この画面を閉じると二度と見られない）

## 2. Apps Script をフォームに設置する

1. 対象の Google フォームを開く
2. 右上の「⋮（その他）」→「スクリプト エディタ」を開く
   （※ 回答先スプレッドシートではなく、**フォーム本体**から開くこと。イベントオブジェクトの形式が異なるため）
3. デフォルトの `Code.gs` の中身を、このディレクトリの `Code.gs` の内容で置き換えて保存

## 3. スクリプト プロパティを設定する

Apps Script エディタ左メニュー「プロジェクトの設定」→「スクリプト プロパティ」で以下を追加：

| プロパティ名 | 値 |
| --- | --- |
| `GITHUB_TOKEN` | 手順1で発行したトークン |
| `GITHUB_REPO` | `Yokochi513/KanSuke` |
| `GITHUB_LABEL` | `フィードバック`（省略可、未設定時のデフォルトも同じ） |

## 4. トリガーを設定する

1. Apps Script エディタ左メニュー「トリガー」（時計アイコン）→「トリガーを追加」
2. 実行する関数: `onFormSubmit`
3. イベントのソース: `フォームから`
4. イベントの種類: `フォーム送信時`
5. 保存時に Google アカウントの権限承認（外部サービスへの接続許可）を求められるので許可する
   - この際「Google hasn't verified this app」という警告が出るが、これは自作の
     Apps Script を自分のアカウントで動かす場合に必ず出る標準の警告であり、
     第三者アプリではないので進めてよい。
     「詳細（Advanced）」→「(プロジェクト名)に移動（安全ではないページ）」→「許可」の順にクリックする。

## 5. 動作確認

フォームからテスト回答を送信し、`Yokochi513/KanSuke` の Issues に
`フィードバック` ラベル付きで新規 Issue が作成されることを確認する。

失敗する場合は Apps Script エディタの「実行数」からエラーログを確認する。
